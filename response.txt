**1. Problem Understanding**

We are given $n$ outposts at sorted integer coordinates and $m$ lanterns at sorted integer coordinates. If we choose an integer radius $s \ge 0$, then every activated lantern illuminates outposts within distance $s$ (i.e., within $[x-s, x+s]$ for lantern coordinate $x$).

Lanterns can only be activated using bursts:

- At most $t$ bursts.
- Each burst activates a contiguous block of lantern indices $[l..r]$ (1-based in the given lantern list).
- Total effort is the sum of lengths of all bursts, $\sum (r-l+1)$, and must be $\le k$.
- Bursts may overlap, but overlap still costs again.

We must output the minimum integer radius $s$ such that all outposts can be illuminated under these constraints.

---

**2. Optimal Approach**

The key observation is that the answer is monotone: if some radius $s$ is large enough to illuminate every outpost, then any larger radius will also work. This lets us binary search on the radius. We search within the range $[0, 2 \cdot 10^9]$, and for each candidate radius we run a feasibility check to determine whether all outposts can be illuminated under the burst and effort constraints.

The feasibility check for a fixed radius works in two stages.

In the first stage, we figure out which lanterns can serve each outpost. Because both arrays are sorted, we sweep through outposts left to right while advancing two pointers over the lantern array. For each outpost at coordinate $a$, we find the contiguous range of lantern indices whose coordinates fall within $[a - s, a + s]$. We call the left end of this range $L$ and the right end $R$. If any outpost has no lantern in range (its $L$ exceeds its $R$), the radius is immediately infeasible.

In the second stage, we use dynamic programming to determine the minimum total effort needed to cover all outposts using at most $t$ bursts. The DP builds up layer by layer: the first layer considers plans using one burst, the second layer adds a second burst, and so on up to $t$ layers. At each layer, for every outpost position $i$, we ask: "If we assign one new burst to cover outposts from some start position $p$ through $i$, what is the cheapest way to do that, given that the outposts before $p$ were already covered by previous bursts?" The cost of a single burst covering outposts $p$ through $i$ depends on how wide the lantern index span must be — specifically, the burst must stretch from the rightmost $L$ value in the group down to the leftmost $R$ value, with a minimum cost of 1.

A naive scan over all possible starting positions $p$ for each $i$ would be quadratic, but we can bring each DP layer down to linear time by exploiting monotonicity. We split candidate start positions into two categories. A "good" start is one where the lantern range already overlaps with the current outpost's range, so the burst costs only 1 unit of effort. A "bad" start is one where there is a gap, and the cost grows with the size of that gap. As we sweep $i$ from left to right, the boundary between good and bad starts only moves forward, so we can track the best good candidate using a monotone deque (which gives us the minimum previous-layer cost in constant time) and the best bad candidate using a simple running minimum. At each outpost we take the better of these two options, giving us the optimal effort for covering up to that outpost in the current layer.

After running all $t$ layers, we check whether the minimum total effort to cover every outpost is within the budget $k$. If so, the candidate radius is feasible; otherwise it is not. The binary search narrows in on the smallest feasible radius and reports it.

**3. Code**
```cpp
#include <algorithm>
#include <deque>
#include <iostream>
#include <vector>

namespace {
const long long kInfinity = (1LL << 60);

/**
 * @brief Computes coverage index intervals [leftIndex, rightIndex] for each outpost.
 *
 * For a given radius, an outpost can be illuminated by lanterns whose coordinates lie within
 * [outpost - radius, outpost + radius]. Because lanternCoordinates is sorted, this set is a
 * contiguous interval in lantern index space.
 *
 * Indices are produced in 1-based lantern index space, and outposts are treated as 1-based
 * in leftIndex/rightIndex to match the DP.
 *
 * @param radius The candidate illumination radius.
 * @param outpostCoordinates Sorted outpost coordinates.
 * @param lanternCoordinates Sorted lantern coordinates.
 * @param leftIndex Output vector (size outpostCount + 1), 1-based outpost indexing.
 * @param rightIndex Output vector (size outpostCount + 1), 1-based outpost indexing.
 * @return True if every outpost has a non-empty interval, false otherwise.
 */
bool computeCoverageIntervals(long long radius,
                              const std::vector<long long>& outpostCoordinates,
                              const std::vector<long long>& lanternCoordinates,
                              std::vector<int>& leftIndex,
                              std::vector<int>& rightIndex) {
    const int outpostCount = static_cast<int>(outpostCoordinates.size());
    const int lanternCount = static_cast<int>(lanternCoordinates.size());

    int leftLanternPointer = 0;   // First lantern with coordinate >= outpost - radius.
    int rightLanternPointer = 0;  // First lantern with coordinate > outpost + radius.

    for (int outpostIndex = 1; outpostIndex <= outpostCount; outpostIndex++) {
        const long long outpostCoordinate = outpostCoordinates[outpostIndex - 1];
        const long long leftCoordinate = outpostCoordinate - radius;
        const long long rightCoordinate = outpostCoordinate + radius;

        while (leftLanternPointer < lanternCount &&
               lanternCoordinates[leftLanternPointer] < leftCoordinate) {
            leftLanternPointer++;
        }

        if (rightLanternPointer < leftLanternPointer) {
            rightLanternPointer = leftLanternPointer;
        }
        while (rightLanternPointer < lanternCount &&
               lanternCoordinates[rightLanternPointer] <= rightCoordinate) {
            rightLanternPointer++;
        }

        leftIndex[outpostIndex] = leftLanternPointer + 1;  // Convert to 1-based lantern index.
        rightIndex[outpostIndex] = rightLanternPointer;    // Count of lanterns <= rightCoordinate.

        if (leftIndex[outpostIndex] > rightIndex[outpostIndex]) {
            return false;
        }
    }
    return true;
}

/**
 * @brief Checks if a given radius can illuminate all outposts within burst and effort limits.
 *
 * Steps:
 * 1) Compute leftIndex/outpostIndex and rightIndex/outpostIndex for the candidate radius.
 * 2) Run DP over up to maxBurstCount bursts to minimize total effort.
 * 3) Return whether the minimal effort to cover all outposts is <= maxEffort.
 *
 * @param radius Candidate radius to test.
 * @param outpostCoordinates Sorted outpost coordinates.
 * @param lanternCoordinates Sorted lantern coordinates.
 * @param maxEffort Maximum allowed total effort (sum of burst lengths).
 * @param maxBurstCount Maximum allowed number of bursts.
 * @return True if feasible, false otherwise.
 */
bool isFeasibleRadius(long long radius,
                      const std::vector<long long>& outpostCoordinates,
                      const std::vector<long long>& lanternCoordinates,
                      int maxEffort,
                      int maxBurstCount) {
    const int outpostCount = static_cast<int>(outpostCoordinates.size());

    std::vector<int> leftIndex(outpostCount + 1, 0);
    std::vector<int> rightIndex(outpostCount + 1, 0);

    if (!computeCoverageIntervals(radius, outpostCoordinates, lanternCoordinates, leftIndex, rightIndex)) {
        return false;
    }

    std::vector<long long> bestEffortAtMostBursts(outpostCount + 1, kInfinity);
    bestEffortAtMostBursts[0] = 0;

    for (int burstNumber = 1; burstNumber <= maxBurstCount; burstNumber++) {
        const std::vector<long long> previousBestEffort = bestEffortAtMostBursts;
        std::vector<long long> bestEffortWithExactlyThisBurstCount(outpostCount + 1, kInfinity);
        bestEffortWithExactlyThisBurstCount[0] = 0;

        int boundaryPointer = 0;
        int processedBadStarts = 0;
        long long bestBadValue = kInfinity;

        std::deque<int> goodDeque;

        for (int endOutpost = 1; endOutpost <= outpostCount; endOutpost++) {
            const int previousCoveredOutposts = endOutpost - 1;
            if (previousBestEffort[previousCoveredOutposts] < kInfinity) {
                while (!goodDeque.empty() &&
                       previousBestEffort[goodDeque.back()] >= previousBestEffort[previousCoveredOutposts]) {
                    goodDeque.pop_back();
                }
                goodDeque.push_back(previousCoveredOutposts);
            }

            while (boundaryPointer < outpostCount &&
                   rightIndex[boundaryPointer + 1] < leftIndex[endOutpost]) {
                boundaryPointer++;
            }

            const int badStartCount = std::min(boundaryPointer, endOutpost);

            while (processedBadStarts < badStartCount) {
                processedBadStarts++;
                const int startOutpost = processedBadStarts;
                const int previousIndex = startOutpost - 1;
                if (previousBestEffort[previousIndex] < kInfinity) {
                    bestBadValue = std::min(
                        bestBadValue,
                        previousBestEffort[previousIndex] - static_cast<long long>(rightIndex[startOutpost])
                    );
                }
            }

            while (!goodDeque.empty() && goodDeque.front() < badStartCount) {
                goodDeque.pop_front();
            }

            long long bestValueForCurrent = kInfinity;

            if (!goodDeque.empty()) {
                bestValueForCurrent = std::min(bestValueForCurrent, previousBestEffort[goodDeque.front()] + 1);
            }

            if (bestBadValue < kInfinity) {
                bestValueForCurrent = std::min(
                    bestValueForCurrent,
                    bestBadValue + static_cast<long long>(leftIndex[endOutpost]) + 1
                );
            }

            bestEffortWithExactlyThisBurstCount[endOutpost] = bestValueForCurrent;
        }

        for (int coveredOutposts = 0; coveredOutposts <= outpostCount; coveredOutposts++) {
            bestEffortAtMostBursts[coveredOutposts] = std::min(
                bestEffortAtMostBursts[coveredOutposts],
                bestEffortWithExactlyThisBurstCount[coveredOutposts]
            );
        }
    }

    return bestEffortAtMostBursts[outpostCount] <= static_cast<long long>(maxEffort);
}
}  // namespace

/**
 * @brief Reads input, binary searches the minimum feasible radius, and prints it.
 *
 * @return Exit code 0 on success.
 */
int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int outpostCount = 0;
    int lanternCount = 0;
    int maxEffort = 0;
    int maxBurstCount = 0;

    std::cin >> outpostCount >> lanternCount >> maxEffort >> maxBurstCount;

    std::vector<long long> outpostCoordinates(outpostCount);
    std::vector<long long> lanternCoordinates(lanternCount);

    for (int outpostIndex = 0; outpostIndex < outpostCount; outpostIndex++) {
        std::cin >> outpostCoordinates[outpostIndex];
    }
    for (int lanternIndex = 0; lanternIndex < lanternCount; lanternIndex++) {
        std::cin >> lanternCoordinates[lanternIndex];
    }

    long long lowRadius = -1;
    long long highRadius = 2000000000LL;

    while (highRadius - lowRadius > 1) {
        const long long midRadius = lowRadius + (highRadius - lowRadius) / 2;
        if (isFeasibleRadius(midRadius, outpostCoordinates, lanternCoordinates, maxEffort, maxBurstCount)) {
            highRadius = midRadius;
        } else {
            lowRadius = midRadius;
        }
    }

    std::cout << highRadius;
    return 0;
}
```

**4. Code Explanation**

The program is organized into three functions: `main`, `computeCoverageIntervals`, and `isFeasibleRadius`.

`main` begins by reading the four parameters ($n$, $m$, $k$, $t$) followed by the two coordinate arrays. It then runs a binary search over the radius. The search starts with `lowRadius = -1` (a value that is always infeasible since radii must be non-negative) and `highRadius = 2000000000` (always feasible since it exceeds the maximum possible distance between any outpost and any lantern). In each iteration, the midpoint is tested via `isFeasibleRadius`. If the midpoint is feasible, the upper bound moves down; otherwise, the lower bound moves up. When the search converges, `highRadius` holds the smallest feasible radius, which is printed.

`computeCoverageIntervals` takes a candidate radius and builds two arrays, `leftIndex` and `rightIndex`, both using 1-based outpost indexing. For each outpost, it determines the contiguous range of lantern indices that fall within distance $s$ of that outpost. It does this by maintaining two pointers that advance through the sorted lantern array: `leftLanternPointer` finds the first lantern at or to the right of $a - s$, and `rightLanternPointer` finds the first lantern strictly beyond $a + s$. The result for outpost $i$ is stored as `leftIndex[i]` (the 1-based index of the first usable lantern) and `rightIndex[i]` (the 1-based index of the last usable lantern). If any outpost ends up with `leftIndex[i] > rightIndex[i]`, the function returns false immediately — that outpost simply cannot be reached by any lantern at this radius.

`isFeasibleRadius` is the core of the solution. After calling `computeCoverageIntervals`, it initializes a DP array `bestEffortAtMostBursts` where entry $i$ holds the minimum total effort needed to illuminate the first $i$ outposts using any number of bursts seen so far. Initially, only position 0 (no outposts to cover) has effort 0; everything else is infinity.

The function then iterates through burst layers from 1 to `maxBurstCount`. In each layer, it computes a new array `bestEffortWithExactlyThisBurstCount` representing the best effort achievable when this layer's burst is the most recently added one. For each ending outpost `endOutpost`, the code considers all possible starting positions for the new burst. A burst covering outposts $p$ through $i$ requires activating a lantern index range wide enough to reach every outpost in that group. The minimum length of such a range depends on whether the starting outpost's rightmost usable lantern ($R_p$) already reaches the ending outpost's leftmost usable lantern ($L_i$).

The optimization works by classifying each candidate start as "good" or "bad". A start $p$ is good if $R_p \ge L_i$, meaning the lantern ranges overlap and the burst costs just 1 unit. A start is bad if $R_p < L_i$, meaning the burst must span a gap and costs $L_i - R_p + 1$. A `boundaryPointer` tracks the dividing line, which only advances as `endOutpost` increases. For good starts, a monotone deque maintains the index with the smallest previous-layer effort, so the best good option is retrieved in constant time. For bad starts, a running minimum of `previousBestEffort[p-1] - rightIndex[p]` is kept in `bestBadValue`, and the total cost for a bad start is `bestBadValue + leftIndex[endOutpost] + 1`. The final answer for each `endOutpost` is the smaller of the best good and best bad options.

After completing each burst layer, the code merges the new results into `bestEffortAtMostBursts` by taking element-wise minimums, ensuring that using fewer bursts remains an option. After all layers are processed, the function checks whether `bestEffortAtMostBursts[outpostCount]` is within the effort budget $k$.

**Sample Walkthrough (Sample 2)**

Input:
```
4 3 2 2
-3 -1 1 3
-2 0 2
```
Here $n=4$, $m=3$, $k=2$, $t=2$. Outposts are at $[-3, -1, 1, 3]$, lanterns at $[-2, 0, 2]$.

The binary search tests candidate radii. Eventually it tests $s = 1$:

*Stage 1 — coverage intervals:* For each outpost, we find which lanterns lie within distance 1.
- Outpost 1 ($a = -3$): range $[-4, -2]$. Lantern 1 ($b = -2$) qualifies. So $L_1 = 1$, $R_1 = 1$.
- Outpost 2 ($a = -1$): range $[-2, 0]$. Lanterns 1 ($b = -2$) and 2 ($b = 0$) qualify. So $L_2 = 1$, $R_2 = 2$.
- Outpost 3 ($a = 1$): range $[0, 2]$. Lanterns 2 ($b = 0$) and 3 ($b = 2$) qualify. So $L_3 = 2$, $R_3 = 3$.
- Outpost 4 ($a = 3$): range $[2, 4]$. Lantern 3 ($b = 2$) qualifies. So $L_4 = 3$, $R_4 = 3$.

All intervals are non-empty, so $s = 1$ is not immediately rejected.

*Stage 2 — DP with up to 2 bursts:*

Layer 1 (one burst available):
- `endOutpost = 1`: The burst must cover outpost 1 alone. Start $p = 1$: cost = $\max(1, L_1 - R_1 + 1) = \max(1, 1) = 1$. Previous effort for covering 0 outposts = 0. Total = $0 + 1 = 1$.
- `endOutpost = 2`: Best single burst covering outposts 1–2: cost = $\max(1, L_2 - R_1 + 1) = \max(1, 1 - 1 + 1) = 1$. Total = $0 + 1 = 1$.
- `endOutpost = 3`: Best single burst covering outposts 1–3: cost = $\max(1, L_3 - R_1 + 1) = \max(1, 2 - 1 + 1) = 2$. Total = $0 + 2 = 2$.
- `endOutpost = 4`: Best single burst covering outposts 1–4: cost = $\max(1, L_4 - R_1 + 1) = \max(1, 3 - 1 + 1) = 3$. Total = $0 + 3 = 3$.

After layer 1: `bestEffortAtMostBursts = [0, 1, 1, 2, 3]`.

Layer 2 (second burst available):
- `endOutpost = 3`: We can split: cover outposts 1–2 with one burst (effort 1 from layer 1), then cover outpost 3 with a second burst. Start $p = 3$: cost = $\max(1, L_3 - R_3 + 1) = 1$. Total = $1 + 1 = 2$. This matches layer 1's value of 2, no improvement.
- `endOutpost = 4`: We can split: cover outposts 1–2 with one burst (effort 1), then cover outposts 3–4 with a second burst. Start $p = 3$: cost = $\max(1, L_4 - R_3 + 1) = \max(1, 3 - 3 + 1) = 1$. Total = $1 + 1 = 2$. This improves over layer 1's value of 3.

After layer 2: `bestEffortAtMostBursts[4] = 2`.

The minimum effort to cover all 4 outposts is 2, which equals $k = 2$, so $s = 1$ is feasible.

The binary search also tests $s = 0$. At $s = 0$, the coverage range for each outpost is just the outpost's own coordinate. Since no lantern sits exactly at $-3$, $-1$, $1$, or $3$, every outpost's interval is empty and `computeCoverageIntervals` returns false immediately. So $s = 0$ is infeasible.

The binary search converges to $s = 1$, which is the output.

**5. Time and Space Complexity**

Let $C = 2 \cdot 10^9$.

- Each feasibility check for a radius:
  - Interval computation: $O(n+m)$
  - DP across at most $t$ bursts: $O(tn)$
  - Total: $O(n+m+tn)$
- Binary search uses $O(\log C)$ feasibility checks.

Overall time complexity:
$$
O(\log C \cdot (n+m+tn)).
$$

Space complexity:
$$
O(n+m)
$$
to store input arrays plus $O(n)$ for DP and interval arrays.

---

**6. Conclusion**

By binary searching the radius and using an $O(n+m+tn)$ feasibility test based on lantern-index intervals and an optimized DP, we can compute the minimum integer radius that allows illuminating all outposts within the burst count and total effort constraints.
